# v10.4 — User position marker on route map

## What's new

When you open the route map, your current GPS position is shown as a
pulsing blue dot with a translucent accuracy circle around it.

Visual: solid blue circle (16px) with white border, Google-Maps-style
pulse animation. The accuracy circle (semi-transparent blue) shows the
GPS uncertainty radius — small for good GPS, large indoors.

## Behaviour

- **Marker appears** when GPS accuracy is ≤ 500m
- **Marker hidden** when accuracy is worse than 500m (better than
  showing a misleading "I'm here" pin in a 1km blob)
- **Updates every 3 seconds** while the map is open (same throttle
  as the route-list marker, no extra battery cost)
- **No auto-pan** when first fix arrives — the map stays fitted to the
  full route. Your dot appears wherever you are within the route view.
  Pan freely without being yanked back.
- **Persists across direction switches** — when you switch tabs (manual
  or auto-switch from v10.3), the map redraws but your marker stays
  put.
- **Removed cleanly** when the map is hidden or the route panel closes

## Why no auto-recenter?

Considered it; decided against it. The route's full extent is what you
opened the map for. If your blue dot is offscreen, panning to find it
takes one swipe — but having the map auto-pan away from the route view
when GPS arrives would be disorienting in the common case where you're
on the route.

If you find yourself wanting auto-recenter, this is a small addition
to revisit later.

## Implementation

- New `addPositionListener` / `removePositionListener` exports in
  `liveLocation.js` — generic pub/sub for position updates
- Position listeners fire BEFORE the route-list accuracy gate, so the
  map can show the accuracy circle even when GPS is too poor for the
  route-list marker
- Map subscribes when opened, unsubscribes when closed
- `renderMap` (which wipes non-tile layers on every render) re-applies
  the user marker from the cached `lastUserPosition` so direction
  switches don't lose the dot
- Late-timer guard: if the user closes the map within 420ms (before
  Leaflet's invalidateSize timeout fires), the deferred startup is
  cancelled

## Caveats

- Tunnels / underground = no GPS = marker freezes at last known
  position. Same hardware limit as v10.2's route-list marker.
- Indoor accuracy is often 100-500m, producing a large accuracy circle
  and a marker that may bounce around. The 500m cutoff hides the
  marker entirely once accuracy gets that bad.
- The pulse animation is disabled under `prefers-reduced-motion`.
