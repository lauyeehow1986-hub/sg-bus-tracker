# v9.1 — Map hang fix

## The bug

Tapping "Show map" on a route hung the UI. Route list rendered correctly
but the map never appeared.

## Root cause

The T6 crowding feature added an async block to the end of `renderRouteList`
that fetched crowding data and then re-rendered to show the dots. But the
re-render triggered the SAME async block, which triggered ANOTHER re-render,
forming a microtask-driven render loop that starved the main thread.

The click event on "Show map" was being processed, but the 420ms timeout it
scheduled never got uninterrupted main-thread time to run Leaflet's
initialisation. Visual effect: the button responded (toggled) but the map
stayed invisible.

## Fix

Two-layer guard in route.js:

1. **Skip-flag**: `renderRouteList` sets `_skipCrowdFetch = true` before the
   re-render and checks it at entry. The re-render skips the async block
   entirely.
2. **Change detection**: before calling `ensureCrowdingFor`, we snapshot the
   cache state for the stations on this route. After the await, compare.
   If nothing changed (everything was already fresh), skip the re-render.
   This makes repeated route views cheap.

Either guard alone would fix the bug; both together mean the path is
safe against future changes to the re-render logic.

## No other changes

All v9 features and behaviour preserved. SW version unchanged; no need
to clear site data — just replace the files.
