# v10.5 — Route list marker now works between stops

## What was wrong (v10.4 and earlier)

The "You are here" marker on the route list used a 100m proximity
threshold — the marker would only appear if you were within 100m of
the closest stop.

In Singapore, bus stops are typically 300-500m apart. When you were
between two stops (e.g. 200m past #43 and 200m before #44), you were
>100m from both. The route list would then show "Not on this route" —
even though you were clearly still on the bus.

The map marker was unaffected — it just shows your raw GPS dot, no
threshold-based logic.

## The fix

Two-tier threshold:

- **≤ 80m** of nearest stop → "**At** stop #43" (you're physically there)
- **80m – 250m** → "**Near** stop #43" (between stops, this is the
  closest reference)
- **250m – 2km** → "Not on this route, closest stop #N is Xm away"
- **> 2km** → silent

The 250m upper bound covers the gap between stops without claiming
you're on the route when you've genuinely walked away from it.

## Visual effect

Marker now stays visible the whole time you're on the bus, including
when between stops. Text changes from "You are here" to
"Nearest (185m)" when you're between stops, so you understand the
marker is showing your closest reference point rather than claiming
you're at that exact stop.

The same wording change applies to the banner: "Near stop #43" instead
of "At stop #43" when you're between stops.

## What this doesn't fix (yet)

When you're approximately equidistant from two adjacent stops (say,
~150m from each), GPS jitter can cause the marker to oscillate between
them every few updates. The 3-second throttle reduces this but doesn't
eliminate it. The proper fix would be tracking sequence progress and
preferring stops you're moving toward — that's a larger change for a
later version if it proves annoying in practice.
